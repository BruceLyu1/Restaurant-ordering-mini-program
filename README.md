# 香港小馆扫码点餐 H5

一个面向香港餐厅场景的扫码点餐 H5 原型。顾客扫码后在手机上浏览菜单、加入购物车并下单；餐厅后台可以管理订单、菜品、分类、桌位、报表、员工、打印和餐厅基础设置。

线上演示：

- 顾客点餐端：[https://brucelyu1.github.io/harbour-ordering-h5/?refresh=21](https://brucelyu1.github.io/harbour-ordering-h5/?refresh=21)
- 餐厅后台：[https://brucelyu1.github.io/harbour-ordering-h5/?view=admin&refresh=21](https://brucelyu1.github.io/harbour-ordering-h5/?view=admin&refresh=21)

## 项目目标

香港不少小型餐厅仍然依赖人工纸笔点餐。本项目用于验证一套低成本扫码点餐流程：

1. 每张桌子放置对应桌号的二维码。
2. 顾客扫码进入点餐页面。
3. 顾客浏览菜单、按分类筛选、加入购物车、确认下单。
4. 餐厅后台按下单顺序接收订单、列印厨房小票、结账。
5. 管理人员查看每日热门菜式和销售报表，用于调整菜单。

## 当前功能

### 顾客端

- 桌号展示，目前演示为 `12号桌`
- 按供应时段展示菜单，例如早餐、午市、晚市
- 菜品分类筛选：饭类、点心、面类、小菜、甜品、饮品等
- 菜品图片、名称、描述、价格展示
- 加入购物车、增减数量、取消误点菜品
- 售罄或不在当前供应时段的菜品不能下单
- 确认下单后展示订单号、金额和状态
- 本桌订单详情可查看待处理、已列印状态
- 后台结账后，顾客端本桌订单自动清空，方便翻台

### 餐厅后台

- **订单管理**
  - 按顾客下单顺序显示订单
  - 区分待处理、已列印、已结账订单
  - 支持手动列印、补印、结账
  - 自动打印开关打开时，新订单会直接进入已列印状态

- **菜单管理**
  - 新增、修改、删除/下架菜品
  - 上传或替换菜品照片
  - 修改菜品名称、价格、分类、供应时段
  - 切换菜品售罄状态
  - 搜索菜品名称
  - 按分类筛选菜品，方便批量管理
  - 分类管理：重命名分类、合并重复分类、删除分类并下架该分类菜品
  - 自动合并简繁重复分类，例如 `饮品` 会归入 `飲品`

- **桌位管理**
  - 查看桌位状态
  - 切换空闲/使用中状态
  - 查看和列印桌码

- **报表分析**
  - 今日、本周、本月、本年营业额
  - 订单总数
  - 售出餐点总数
  - 菜品销量排行
  - 菜品销售额排行
  - 右侧热门菜式可跳转到完整报表

- **员工管理**
  - 新增员工
  - 启用或停用账号
  - 用于后续正式权限系统接入

- **打印设置**
  - 选择厨房热敏打印机
  - 设置每张订单份数
  - 开关自动打印新订单
  - 开关新订单提示声
  - 打印测试小票

- **餐厅设置**
  - 修改餐厅名称、电话、地址
  - 设置默认语言
  - 设置早餐、午市、晚市供应时间

## 菜单数据

当前默认菜单共 60 款：

- 饭类：10 款
- 点心：10 款
- 面类：10 款
- 小菜：10 款
- 甜品：10 款
- 饮品：10 款

菜品图片目前复用本地菜品图集，正式商用时可由餐厅后台逐个上传真实照片。

## 技术栈

- React
- Vite
- CSS 原生样式
- GitHub Pages 静态部署
- 浏览器 `localStorage` 模拟数据持久化

当前版本是前端 MVP，所有订单、菜单、设置、员工和桌位数据都保存在浏览器本地，用于快速演示完整业务流程。

## 当前限制

当前版本主要用于原型演示，还不是正式商用系统：

- 数据保存在当前浏览器，换设备后不会同步
- 没有真实员工登录和权限控制
- 没有真实后端数据库
- 没有接入真实打印机驱动或 ESC/POS 打印服务
- 没有正式二维码批量生成和桌号绑定后台
- 没有在线支付、会员、优惠券和外卖功能

## 后续建议

正式商用版本建议接入：

- Supabase PostgreSQL：保存餐厅、菜单、订单、桌台、员工等数据
- Supabase Auth：员工登录、角色权限、账号停用
- Supabase Storage：菜品图片上传与管理
- Supabase Realtime：顾客下单后后台实时出单
- ESC/POS 打印服务：对接厨房热敏小票机
- 二维码生成：按餐厅和桌号生成专属点餐链接
- 多餐厅隔离：让系统可服务多家餐厅
- 数据备份、日志、监控和后台部署
