# 香港小馆扫码点餐 H5

更新日期：2026-06-25

这是一个面向香港中小餐厅的扫码点餐 H5 原型。顾客扫码后可以在手机上浏览菜单、加入购物车并下单；餐厅后台可以接收订单、打印厨房小票、结账、维护菜单、管理桌位并查看销售报表。

当前版本仍是前端演示系统，数据保存在浏览器本地 `localStorage`。项目已经完成前端架构重构、TypeScript 迁移、Zustand 状态层和轻量 i18n，为下一步接入真实数据库做准备。

## 当前功能

### 顾客端

- 根据 URL 参数识别桌号，例如 `?view=guest&table=02`。
- 按分类浏览菜单，支持推荐、饭类、点心、面类、小菜、甜品、饮品等分类。
- 展示菜品图片、名称、描述、价格和售罄状态。
- 支持加入购物车、增减数量、删除菜品和确认下单。
- 下单后显示订单号、桌号、金额和订单状态。
- 可以查看本桌未结账订单详情。
- 后台结账后，本桌订单会从顾客端清空。
- 顾客端支持繁体中文 / 英文 UI 切换，切换只影响当前浏览器界面。

### 餐厅后台

- 运营总览：查看待处理订单、营业额、桌位和菜单概况。
- 订单管理：查看新订单、打印、补印、结账。
- 菜单管理：新增、编辑、删除/下架菜品，维护图片、名称、描述、分类、价格、供应时段和售罄状态。
- 分类管理：新增、重命名、合并重复分类、删除分类并下架相关菜品。
- 桌位管理：查看桌位状态、复制点餐链接、打印桌码。
- 报表分析：查看日/周/月/年营收、订单数、售出份数和菜品排行。
- 员工管理：新增员工、设置角色、启用或停用账号。
- 打印设置：选择模拟打印机、设置份数、自动打印和提示声。
- 餐厅设置：维护餐厅名称、电话、地址、默认语言和供应时段。

## 代码结构

```text
src/
  components/   可复用 UI、顾客端组件、后台组件
  data/         演示用种子数据
  i18n/         轻量翻译表、语言上下文、日期格式化
  pages/        顾客端和后台各页面
  services/     本地存储服务层，未来数据库替换点
  stores/       Zustand 前端业务状态
  types/        TypeScript 业务类型
  utils/        金额、日期、分类、订单、桌位、图片等纯工具
```

## 推荐验收流程

1. 打开顾客端，例如 `?view=guest&table=02`。
2. 添加几道菜到购物车。
3. 确认下单。
4. 切换到后台订单管理。
5. 确认后台能看到新订单。
6. 点击打印，确认订单状态变为已打印。
7. 回到顾客端订单详情，确认状态同步变化。
8. 后台点击结账。
9. 回到顾客端，确认本桌订单清空。
10. 查看桌位状态和报表数据是否同步变化。
11. 在后台菜单管理新增/编辑菜品描述，确认顾客端能显示。
12. 切换繁体中文 / 英文 UI，确认系统文案变化且业务数据不被翻译。

## 已完成的重构

- 将原来的巨型文件拆分为 `data`、`utils`、`services`、`stores`、`components` 和 `pages`。
- 将源码迁移到 TypeScript，并补充核心业务类型。
- 建立 service 层，页面不直接访问 `localStorage`。
- 引入 Zustand 收口菜单、订单和设置状态。
- 增加测试覆盖金额、订单、分类、服务层、store、页面和 i18n。
- 修复生产构建 React 版本、入口文件、错误边界和 Windows 本地构建兼容问题。
- 增加轻量 i18n，不引入第三方国际化框架。
- 菜品描述支持后台编辑并在顾客端展示。

## 当前限制

- 暂未接入真实数据库。
- 暂未实现员工登录、权限和 RLS。
- 暂未接入真实打印机或 ESC/POS 打印服务。
- 菜品图片上传后仍保存在当前浏览器本地。
- 不同设备之间暂时不会同步订单和菜单。
- 暂未做多门店、多餐厅数据隔离。
- 暂未接入在线支付、会员、优惠券和外卖功能。

## 下一步建议

第一优先级是接入真实数据库，建议从 Supabase 开始：

- PostgreSQL：保存餐厅、桌位、菜单、订单、员工和设置。
- Auth：员工登录和权限控制。
- Storage：保存菜品图片。
- Realtime：同步顾客端和后台订单状态。
- RLS：隔离餐厅和门店数据。

接入数据库后，系统就可以从“本地演示原型”升级为“多设备可试用版本”。
