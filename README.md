# 香港小馆扫码点餐 H5

更新日期：2026-06-23

这是一个面向香港中小餐厅的扫码点餐 H5 原型。顾客扫描桌面二维码后，可以在手机上浏览菜单、加入购物车、确认下单；餐厅后台可以接收订单、列印厨房小票、结账、管理菜品、查看桌位状态和分析销售数据。

当前项目定位是“可演示的前端 MVP”。它已经可以跑通从顾客扫码点餐到后台处理订单的核心流程，但还没有接入真实后端数据库、员工登录、云端图片存储和真实打印机服务。

## 项目最初需求

香港很多小型餐厅仍然依赖人工纸笔点餐。目标是做一套低成本扫码点餐系统：

1. 每张桌放一个二维码，二维码对应桌号。
2. 顾客扫码后进入对应桌号的手机点餐页面。
3. 顾客查看菜品、加入购物车、确认下单。
4. 餐厅后台看到是哪张桌点了什么，并按下单顺序出单。
5. 前台可以列印、结账，顾客用餐结束后到前台买单。
6. 后台统计每天菜品销量，帮助餐厅判断哪些菜更受欢迎。
7. 系统可以适用于不同餐厅，餐厅只需要维护自己的菜品、价格、图片和桌位。

## 截至目前已完成

### 1. 顾客点餐端

- 手机 H5 点餐页面。
- 根据 URL 参数识别桌号，例如 `?view=guest&table=02`。
- 菜品按分类展示：推荐、饭类、点心、面类、小菜、甜品、饮品。
- 推荐分类根据顾客累计点餐量排序。
- 菜品显示真实菜品图片、名称、描述和价格。
- 支持加入购物车、增加数量、减少数量、删除误点菜品。
- 下单前可以打开购物车确认明细。
- 确认下单后生成订单号，并显示桌号和金额。
- 顾客端顶部有“订单详情”，可以随时查看本桌订单明细。
- 订单详情会显示待处理、已列印、已结账等状态。
- 后台结账后，顾客端本桌订单会清空，方便下一批顾客使用。
- 售罄菜品和非当前供应时段菜品不能下单。

### 2. 餐厅后台

后台包含以下管理模块：

- 营运总览
- 订单管理
- 菜单管理
- 桌位管理
- 报表分析
- 员工管理
- 打印设置
- 餐厅设置

### 3. 订单管理

- 后台按顾客下单顺序显示订单。
- 区分待处理、已列印、已结账订单。
- 支持手动列印和补印。
- 支持结账。
- 打印设置里打开“自动打印新订单”后，新订单会直接进入已列印状态。
- 后台订单右侧显示今日热门菜式。
- 订单管理旁边的红色数字已改为真实待处理订单数量，待处理为 0 时不再显示。

### 4. 菜单管理

- 默认菜单已扩展到 60 款菜品。
- 每个分类约 10 款菜，包括饭类、点心、面类、小菜、甜品、饮品。
- 菜品图片已换成本地真实菜品图片资源。
- 支持新增菜品。
- 支持修改菜品名称、价格、图片、分类和供应时段。
- 支持删除或下架菜品。
- 支持切换售罄状态。
- 支持按菜品名称搜索。
- 支持按分类筛选菜品，方便后台人员批量维护。
- 新增菜品时可以选择已有分类，也可以新增分类。
- 支持分类管理：重命名分类、合并重复分类、删除分类并下架该分类菜品。
- 自动合并简繁重复分类，例如 `饮品` 会归入 `飲品`。

### 5. 桌位管理与二维码

- 桌位管理可以查看所有桌位。
- 每张桌都可以查看对应的点餐链接。
- 每张桌都可以生成二维码桌码。
- 二维码弹窗支持复制链接和列印桌码。
- 顾客扫码后会进入对应桌号的顾客点餐页面。
- 如果某桌有未结账订单，桌位自动显示为用餐中，颜色为红色。
- 如果某桌没有未结账订单，桌位显示为空桌，颜色为绿色。
- 后台结账后，该桌自动恢复为空桌。

### 6. 报表分析

- 支持今日营业额。
- 支持本周营业额。
- 支持本月营业额。
- 支持本年营业额。
- 显示订单总数。
- 显示售出餐点总数。
- 显示菜品销量排行。
- 显示菜品销售额排行。
- 右侧“热门菜式”可以跳转到完整报表。
- 如果今天没有订单，热门菜式为空；有订单后会按当天销量实时排序。

### 7. 员工管理

- 支持新增员工。
- 支持设置员工角色，例如楼面、收银员、经理。
- 支持启用或停用员工账号。
- 当前还没有接入真实登录系统，员工管理主要用于展示后续权限系统方向。

### 8. 打印设置

- 支持选择模拟打印机。
- 支持设置每张订单打印份数。
- 支持自动打印新订单开关。
- 支持新订单提示声开关。
- 支持打印测试小票。
- 自动打印和手动列印不冲突：自动打印负责新订单首次出单，手动列印用于人工确认和补印。

### 9. 餐厅设置

- 支持修改餐厅名称、电话、地址。
- 支持设置默认语言。
- 支持设置早餐、午市、晚市供应时间。
- 菜单管理里的菜品供应时段会结合这里的营业时段生效。

## 功能之间的关系

### 桌码和顾客端

每张桌的二维码本质上是一个带桌号参数的链接。顾客扫码进入后，系统会自动识别桌号。后台收到订单时，也会显示对应桌号。

### 菜单管理和顾客端菜单

后台菜单管理决定顾客端看到的菜品内容，包括图片、名称、价格、分类、供应时段和售罄状态。

### 顾客下单和订单管理

顾客确认下单后，后台订单管理会出现新订单。订单按照提交时间排序，厨房可以按顺序准备。

### 打印设置和订单状态

自动打印关闭时，新订单是待处理，需要后台手动点列印。自动打印打开时，新订单会直接变成已列印。顾客端订单详情会显示状态变化。

### 结账和桌位状态

后台结账后，该订单进入已结账状态。顾客端本桌订单会清空，桌位状态会从用餐中变为空桌。

### 订单和报表

报表数据来自订单。菜品销量、销售额、日周月年营业额都会根据订单自动统计。

### 推荐和热门菜式

- 顾客端“推荐”按累计销量排序。
- 后台右侧“热门菜式”按今日销量排序。

两者都来自订单数据，但统计口径不同。

## 当前限制

当前版本主要用于演示和流程验证，还不是正式商用系统：

- 数据保存在当前浏览器，换设备后不会同步。
- 没有真实后端数据库。
- 没有真实员工登录和权限控制。
- 菜品图片上传后只保存在当前浏览器本地。
- 没有接入真实打印机驱动或 ESC/POS 打印服务。
- 没有多餐厅、多门店数据隔离。
- 没有正式订单历史后台和数据备份。
- 暂时不做在线支付、会员、优惠券和外卖功能。

## 接下来建议做什么

### 第一优先级：接入真实后端

建议使用 Supabase：

- PostgreSQL 保存餐厅、桌位、菜单、订单、员工和设置数据。
- Auth 做员工登录和权限控制。
- Storage 保存菜品图片。
- Realtime 做顾客端和后台订单状态实时同步。

这一步完成后，系统就可以从“本地演示”升级为“多设备可试用”。

### 第二优先级：真实订单流程

- 顾客下单写入数据库。
- 后台实时收到订单。
- 后台列印后，顾客端订单状态实时变更。
- 后台结账后，顾客端本桌订单实时清空。
- 桌位状态实时变更。
- 增加订单历史查询和日结统计。

### 第三优先级：真实打印服务

- 对接 ESC/POS 热敏打印机。
- 支持厨房小票格式配置。
- 支持自动打印失败后的补印。
- 支持按档口或分类分单打印。

### 第四优先级：餐厅试用准备

- 找 1 到 3 家香港小型餐厅试用。
- 录入真实菜单。
- 上传真实菜品照片。
- 设置真实桌位数量。
- 打印每张桌的二维码。
- 培训前台如何接单、列印、结账。
- 收集顾客和员工反馈。

### 暂不优先做

- 在线支付
- 会员系统
- 优惠券
- 外卖
- 复杂库存系统
- 多门店 SaaS 管理后台

这些功能可以放到真实点餐、接单、结账流程稳定之后再做。

## 推荐测试流程

1. 打开顾客端指定桌号链接，例如 `?view=guest&table=02`。
2. 确认页面显示 02 号桌。
3. 切换菜单分类。
4. 加入几道菜到购物车。
5. 打开购物车确认数量和金额。
6. 点击确认下单。
7. 打开顾客端“订单详情”查看订单。
8. 切换到餐厅后台。
9. 在订单管理里查看新订单。
10. 点击列印，确认订单变成已列印。
11. 回到顾客端订单详情，确认状态变化。
12. 后台点击结账。
13. 回到顾客端，确认本桌订单清空。
14. 进入桌位管理，确认该桌变为空桌。
15. 进入报表分析，确认销量和营业额变化。

## 当前结论

项目目前已经完成了扫码点餐系统最核心的产品原型：顾客点餐、后台接单、列印、结账、桌位状态、二维码桌码、菜单维护、分类管理、供应时段、销量报表和基础设置。

下一步真正关键的不是继续堆更多前端页面，而是接入真实数据库、登录、图片存储和实时同步。只要这一步完成，就可以开始找第一家餐厅做小范围真实试用。
