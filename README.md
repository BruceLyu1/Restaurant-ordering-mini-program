# 海港小馆扫码点餐 H5

更新日期：2026-06-29

这是一个面向香港中小餐厅的扫码点餐 H5 应用。顾客扫码后可以在手机上浏览菜单、加入购物车、填写单品备注并下单；餐厅后台可以接收订单、打印/补印、结账、维护菜单、管理桌位和员工，并查看营业报表。

当前版本仍是前端演示系统，数据保存在浏览器本地 `localStorage`。项目已经完成前端模块化重构、TypeScript 迁移、Zustand 状态层、轻量 i18n、核心测试补强和真实餐厅试点前的若干质量修复，为下一步接入真实数据库做准备。

## 当前功能

### 顾客端

- 根据 URL 参数识别桌号，例如 `?view=guest&table=02`。
- 按分类浏览菜单，支持推荐、饭类、点心、面类、小菜、甜品、饮品等分类。
- 展示菜品图片、名称、描述、价格、售罄状态和供应时段。
- 支持加入购物车、增减数量、删除菜品、填写单品备注和确认下单。
- 下单后展示订单号、桌号、金额、订单状态和单品备注。
- 可查看本桌未结账订单详情，后台结账后顾客端同步清空。
- 支持繁体中文 / 英文 UI 切换，切换只影响当前浏览器界面，不翻译餐厅自定义业务数据。

### 餐厅后台

- 运营总览：查看待处理订单、营业额、桌位和菜单概况。
- 订单管理：查看新订单、打印、补印、结账，并显示顾客填写的单品备注。
- 菜单管理：新增、编辑、删除/下架菜品，维护图片、名称、描述、分类、价格、供应时段和售罄状态。
- 分类管理：新增、重命名、合并重复分类、删除分类并下架相关菜品。
- 桌位管理：查看桌位状态、复制点餐链接、打印桌码。
- 报表分析：查看日/周/月/年营收、订单数、售出份数和菜品排行。
- 员工管理：新增员工、设置角色、启用或停用账号。
- 打印设置：选择模拟打印机、设置份数、自动打印和提示声。
- 餐厅设置：维护餐厅名称、电话、地址、默认语言和供应时段。

## 技术栈

- React 19
- Vite
- TypeScript
- Zustand
- Vitest
- Testing Library
- 原生 CSS
- `localStorage` 本地演示数据持久化

## 代码结构

```text
src/
  components/   可复用 UI、顾客端组件、后台组件
  data/         演示用种子数据
  i18n/         轻量翻译表、语言上下文、日期格式化
  pages/        顾客端和后台各页面
  services/     本地存储服务层，未来数据库替换点
  stores/       Zustand 前端业务状态
  styles/       按业务域拆分的 CSS
  types/        TypeScript 业务类型
  utils/        金额、日期、分类、订单、桌位、图片等纯工具
```

## 本地运行

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

常用访问地址：

```text
http://127.0.0.1:5173/?view=guest
http://127.0.0.1:5173/?view=admin
```

端口可能被 Vite 自动调整，请以终端实际输出为准。

运行测试：

```bash
npm test
```

TypeScript 检查：

```bash
npx tsc --noEmit
```

生产构建：

```bash
npm run build
```

## 推荐验收流程

1. 打开顾客端，例如 `?view=guest&table=02`。
2. 添加几道菜到购物车，并给其中一道菜填写备注。
3. 确认下单。
4. 切换到后台订单管理，确认后台能看到新订单和备注。
5. 点击打印，确认订单状态变为已打印。
6. 回到顾客端订单详情，确认状态同步变化。
7. 后台点击结账。
8. 回到顾客端，确认本桌订单清空。
9. 查看桌位状态和报表数据是否同步变化。
10. 在后台菜单管理新增或编辑菜品描述，确认顾客端能显示。
11. 切换繁体中文 / 英文 UI，确认系统文案变化且业务数据不被翻译。

## 已完成优化

- 将原有巨型文件拆分为 `data`、`utils`、`services`、`stores`、`components` 和 `pages`。
- 源码迁移到 TypeScript，并补齐核心业务类型。
- 建立 service 层，页面和组件不直接访问 `localStorage`，方便后续替换为真实数据库。
- 引入 Zustand 收口菜单、订单、设置、员工和桌位状态。
- 增加轻量 i18n，不引入第三方国际化框架。
- 菜品描述支持后台编辑并在顾客端展示。
- 增加订单备注能力，顾客端、顾客订单历史和后台订单卡片均可显示备注。
- 修复生产构建入口、React 依赖 shim、错误边界、用户可见乱码和 Windows 本地构建兼容问题。
- 修复下单后同一订单被重复显示的问题。
- 拆分 CSS 文件，提高后续维护性但保持现有视觉不变。
- 补强服务层、store、页面、组件、报表、存储、i18n 和错误边界测试。

## 当前限制

- 暂未接入真实数据库。
- 暂未实现员工登录、权限控制和 RLS。
- 暂未接入真实打印机或 ESC/POS 打印服务。
- 菜品图片上传后仍保存在当前浏览器本地。
- 不同设备之间暂时不会同步订单、菜单和设置。
- 暂未做多门店、多餐厅数据隔离。
- 暂未接入在线支付、会员、优惠券和外卖功能。

## 下一步建议

第一优先级是接入真实数据库，建议从 Supabase 开始：

- PostgreSQL：保存餐厅、桌位、菜品、订单、员工和设置。
- Auth：员工登录和权限控制。
- Storage：保存菜品图片。
- Realtime：同步顾客端和后台订单状态。
- RLS：隔离餐厅和门店数据。

接入数据库后，系统就可以从“本地演示原型”升级为“多设备可试用版本”，再逐步补真实打印、权限、营收报表和餐厅上线流程。
